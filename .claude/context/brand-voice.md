# cside Brand Voice Guide

## Voice Pillars

### 1. Technical & Credible
- Lead with data, specifics, and technical accuracy
- Reference standards (PCI DSS 4.0.1, NIST, OWASP) by name
- Use precise terminology: "client-side scripts," "third-party tags," "DOM manipulation"
- Back claims with metrics where possible

### 2. Direct & Confident
- Short, declarative sentences preferred
- No hedge words: avoid "might," "could potentially," "it's possible that"
- State capabilities plainly: "cside detects malicious scripts in real-time"
- Use active voice 90%+ of the time

### 3. Security-First
- Frame everything through the lens of protection and risk reduction
- Acknowledge threats without fearmongering
- Position cside as the expert, not the alarmist
- Use "protect," "detect," "prevent," "monitor" — not "scary," "dangerous," "terrifying"

### 4. Compliance-Focused
- Treat compliance as a business enabler, not a burden
- Reference specific requirements (e.g., PCI DSS 4.0.1 Requirement 6.4.3)
- Connect compliance to business outcomes: reduced fines, maintained trust, continued operations

## Tone Guidelines

| Context | Tone |
|---------|------|
| Blog posts | Informative, educational, expert peer-to-peer |
| Landing pages | Confident, benefit-driven, action-oriented |
| Compare pages | Factual, fair, differentiation-focused |
| Technical docs | Precise, structured, no fluff |
| Social media | Accessible, topical, engagement-focused |
| Email | Personal, value-first, concise |

## Messaging Framework

### Primary Message
cside provides real-time visibility and protection for every script running on your website.

### Supporting Messages
1. **Compliance**: Meet PCI DSS 4.0.1 requirements for client-side security with automated monitoring
2. **Protection**: Detect and block malicious scripts before they steal customer data
3. **Visibility**: Know exactly what third-party code is running on your site and what it's doing
4. **Speed**: Deploy in minutes, not months -- no code changes required

## Audience Understanding

### Primary: Security/IT Leaders (CISO, VP Security, Security Engineers)
- Care about: threat detection, compliance, integration with existing stack
- Language: technical, standards-referenced, risk-quantified

### Secondary: Compliance Officers / GRC Teams
- Care about: audit readiness, documentation, regulatory mapping
- Language: regulation-specific, evidence-focused, process-oriented

### Tertiary: E-commerce / Digital Leaders (CTO, VP Digital, Head of E-commerce)
- Care about: revenue protection, customer trust, site performance impact
- Language: business outcomes, ROI, competitive advantage

## Words We Use / Don't Use

| Use | Don't Use |
|-----|-----------|
| protect | secure (as verb, overused) |
| detect | discover |
| client-side security | web security (too generic) |
| third-party scripts | third-party code (less specific) |
| real-time | instant (less precise) |
| cside | C/Side, CSide, C-Side, Cside |
| e-skimming | eSkimming, E-Skimming |
| Magecart | magecart, MageCart |
